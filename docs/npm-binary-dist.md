# Rust CLI → NPM Binary Distribution

Reusable architecture for distributing Rust CLI binaries as npm packages with platform-specific subpackages. Works locally — no GitHub Actions required.

## Workflow

```
just build-all          # Build all 5 platforms into target/
bun scripts/npm-pack.ts # Package built binaries into npm/<label>/
cd npm/darwin-arm64 && npm publish   # Manual per-platform publish
cd ../linux-x64       && npm publish
...
cd .. && npm publish                          # Publish main package
```

## Architecture

```
Project root/
├── Cargo.toml                    # workspace + binary crate
├── justfile                      # build-release-*, build-all, pack
├── scripts/
│   └── npm-pack.ts               # Generic Bun packager (copy to any project)
└── npm/                          # Generated by npm-pack.ts + static files
    ├── package.json              # Static — tracked in git, version bumped by script
    ├── binary.js                 # Static resolver — tracked in git
    └── <label>/                  # Generated — gitignored (binaries)
        ├── package.json
        └── bin/talon
```

## Per-project setup (3 steps)

### 1. Copy `scripts/npm-pack.ts`

Copy the Bun script to your project. No modifications needed.

### 2. Add Justfile targets

```just
build-release-darwin-arm64:
    cargo build -p <pkg> --release --target aarch64-apple-darwin --locked

build-release-darwin-x64:
    PATH="$HOME/.cargo/bin:$PATH" "$HOME/.cargo/bin/cargo" zigbuild -p <pkg> --release --target x86_64-apple-darwin --locked

build-release-linux-x64:
    PATH="$HOME/.cargo/bin:$PATH" "$HOME/.cargo/bin/cargo" zigbuild -p <pkg> --release --target x86_64-unknown-linux-gnu --locked

build-release-linux-arm64:
    PATH="$HOME/.cargo/bin:$PATH" "$HOME/.cargo/bin/cargo" zigbuild -p <pkg> --release --target aarch64-unknown-linux-gnu --locked

build-release-win32-x64:
    PATH="$HOME/.cargo/bin:$PATH" "$HOME/.cargo/bin/cargo" zigbuild -p <pkg> --release --target x86_64-pc-windows-gnu --locked

build-all:
    just build-release-darwin-arm64
    just build-release-darwin-x64
    just build-release-linux-x64
    just build-release-linux-arm64
    just build-release-win32-x64

pack:
    bun scripts/npm-pack.ts --npm-org <org>

pack-no-smoke:
    bun scripts/npm-pack.ts --npm-org <org> --skip-smoke
```

Replace `<pkg>` with your Cargo package name, `<org>` with your NPM org.

### 3. Install toolchain

```bash
rustup target add aarch64-apple-darwin x86_64-apple-darwin \
                 x86_64-unknown-linux-gnu aarch64-unknown-linux-gnu \
                 x86_64-pc-windows-gnu
cargo install cargo-zigbuild
```

## Static files to create

### `npm/package.json` (tracked in git)

```json
{
  "name": "@<org>/<binary>",
  "version": "0.1.0",
  "private": false,
  "type": "module",
  "bin": { "<binary>": "./binary.js" },
  "optionalDependencies": {
    "@<org>/<binary>-darwin-arm64": "0.1.0",
    "@<org>/<binary>-darwin-x64": "0.1.0",
    "@<org>/<binary>-linux-x64": "0.1.0",
    "@<org>/<binary>-linux-arm64": "0.1.0",
    "@<org>/<binary>-win32-x64": "0.1.0"
  }
}
```

Version and optionalDependencies versions are bumped by `npm-pack.ts`.

### `npm/binary.js` (tracked in git)

Node.js shim that resolves the platform-specific binary from node_modules:

```js
#!/usr/bin/env node
const { createRequire } = require("node:module");
const require = createRequire(import.meta.url);

const PLATFORMS = {
  darwin: { arm64: "@<org>/<binary>-darwin-arm64", x64: "@<org>/<binary>-darwin-x64" },
  linux: { arm64: "@<org>/<binary>-linux-arm64", x64: "@<org>/<binary>-linux-x64" },
  win32: { x64: "@<org>/<binary>-win32-x64" },
};

const platform = process.platform;
const arch = process.arch === "arm64" ? "arm64" : "x64";
const pkg = PLATFORMS[platform]?.[arch];
if (!pkg) { console.error(`unsupported platform: ${platform}-${arch}`); process.exit(1); }

try {
  const binary = require.resolve(`${pkg}/bin/talon`);
  process.execPath = binary;
  require("child_process").execFileSync(binary, process.argv.slice(2), { stdio: "inherit" });
} catch (err) {
  console.error(`could not resolve ${pkg}: ${err.message}`);
  process.exit(1);
}
```

### `.gitignore`

Add: `npm/*/bin/*`

## How `npm-pack.ts` works

The script reads **everything** from Cargo.toml via `cargo metadata`:

| Cargo.toml field | npm package.json field |
|-------------------|----------------------|
| `[package.name]` | used to derive binary name (strips `-cli`, `_cli`, `.bin`) |
| `[package.version]` | version on all packages + bumps static main package.json |
| `[package.description]` | logged for reference |
| `[package.license]` | license on platform packages |
| `[package.repository]` | repository URL |
| `[package.homepage]` | logged for reference |

The only required CLI arg is `--npm-org <org>` — everything else is auto-detected.

Binary name derivation: strips `-cli`, `_cli`, or `.bin` suffixes from the Cargo package name. Override with `--binary <name>`.

## Environment overrides

| Env var | Purpose | Default |
|---------|---------|---------|
| `NPM_ORG` | NPM org prefix (alternative to `--npm-org`) | required |
| `TALON_SKIP_SMOKE` | Skip smoke tests | `0` |
| `TALON_REQUIRE_TARGET_SMOKE` | Fail without cross-platform runtime | `0` |
| `TALON_MAX_BINARY_BYTES` | Max binary size | 30MB |

## How it works

1. **Justfile builds** — Each platform target compiles independently into `target/<triple>/release/`. No cross-build aggregation step.

2. **Script packages** — `npm-pack.ts` reads already-built binaries from the cargo target directory, copies them into `npm/<label>/bin/`, and generates package.json files for each platform subpackage. It also bumps the version in the static `npm/package.json`.

3. **Platform packages** — Each target gets its own npm package with `os` + `cpu` constraints. npm resolves the right one automatically during install. No postinstall scripts.

4. **Binary resolver** — `binary.js` uses `require.resolve()` to find the platform-specific binary in `node_modules`. Simple, no TS compilation needed.

5. **Smoke testing** — Runs `--version` on each binary. Native targets run directly; Linux ARM uses qemu; Windows uses wine. Skippable with `--skip-smoke`.

## Publishing (manual)

Publishing is intentionally not automated. After `just pack`:

```bash
cd npm/darwin-arm64 && npm publish [--dry-run]
cd ../darwin-x64    && npm publish [--dry-run]
cd ../linux-x64     && npm publish [--dry-run]
cd ../linux-arm64   && npm publish [--dry-run]
cd ../win32-x64     && npm publish [--dry-run]
cd ..               && npm publish [--dry-run]
```

This gives you control over the order and ability to inspect each package before publishing.

## Known limitations

- **Darwin x64 cross-compile** requires Zig (no native macOS toolchain in container/CI)
- **Windows smoke test** requires Wine installed
- **No musl support** — targets are glibc-based
- **Single binary crate** — if workspace has multiple bin crates, specify `--package <name>`

## Alternatives considered

| Tool | Why not |
|------|---------|
| `cargo-dist` | Generates its own CI config, opinionated about GitHub Releases, heavier setup |
| Shell script | Harder to maintain; Bun gives better error handling and type safety |
| NAPI | For Node.js native addons (.node files), not standalone binaries |
| TypeScript shim | Unnecessary compilation step — plain JS resolver works fine |
